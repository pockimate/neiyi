## 功能模块

### 1. 📊 仪表板 (Dashboard)
**路径**: `/admin/dashboard`

**功能**:
- 实时统计数据
  - 总收入: $12,426 (+12.5%)
  - 总订单: 156 (+8.2%)
  - 总产品: 12
  - 总客户: 89 (+15.3%)
- 最近订单列表（4条）
- 热销产品排行（4个）
- 销售趋势图表（12个月）

**特色**:
- 彩色统计卡片
- 订单状态标签
- 交互式图表
- 快速链接到详情页

### 2. 📦 产品管理 (Products)
**路径**: `/admin/products`

**功能**:
- 产品列表展示
  - 产品图片
  - 名称和描述
  - 分类
  - 价格（原价/折扣价）
  - 库存数量
  - 状态标签
- 搜索产品（按名称/描述）
- 按分类筛选
- 添加新产品
- 编辑产品信息
- 删除产品

**产品表单字段**:
- 产品名称 *
- 描述 *
- 价格 *
- 原价（可选）
- 分类 * (Bras & Sets, Bodysuits, Chemises, Corsets)
- 库存 *
- 标签（可选：New, Sale, Bestseller）

### 3. 📋 订单管理 (Orders)
**路径**: `/admin/orders`

**功能**:
- 订单列表展示
  - 订单号
  - 客户信息（姓名、邮箱）
  - 订单日期
  - 商品数量
  - 订单总额
  - 订单状态
- 按状态筛选（Pending, Processing, Shipped, Completed, Cancelled）
- 搜索订单（按订单号/客户名/邮箱）
- 查看订单详情
- 更新订单状态（下拉选择）

**订单详情包含**:
- 客户信息
- 配送地址
- 订单商品列表
- 价格明细（小计、税费、总计）

### 4. 🎟️ 优惠码管理 (Coupons)
**路径**: `/admin/coupons`

**功能**:
- 优惠码卡片展示
  - 优惠码
  - 描述
  - 折扣百分比
  - 使用次数/限制
  - 过期日期
  - 状态（Active/Expired）
- 按状态筛选
- 添加新优惠码
- 编辑优惠码
- 删除优惠码

**优惠码表单字段**:
- 优惠码 * (大写字母)
- 描述 *
- 折扣百分比 * (1-100)
- 使用限制（可选，留空为无限制）
- 过期日期 *

**预设优惠码**:
- `SAVE10` - 10% off (45/100 已使用)
- `SAVE20` - 20% off (23/50 已使用)
- `WELCOME15` - 15% off (67/∞ 已使用)
- `SUMMER25` - 25% off (已过期)

## 技术实现

### 认证系统
- 使用 LocalStorage 存储登录状态
- 路由守卫（未登录自动跳转到登录页）
- 演示账号验证
- 退出登录清除状态

### 数据管理
- 使用 Vue 3 Composition API
- Reactive 响应式数据
- Computed 计算属性
- 实时筛选和搜索

### 组件结构
```
pages/admin/
```

## 使用流程

### 1. 登录后台
1. 访问 http://localhost:3000/admin/login
2. 输入演示账号
3. 点击 "Sign In"

### 2. 查看仪表板
- 自动跳转到 Dashboard
- 查看统计数据和图表
- 点击 "View All" 查看详情

### 3. 管理产品
1. 点击侧边栏 "Products"
2. 使用搜索框或分类筛选
3. 点击 "Add Product" 添加新产品
4. 点击编辑图标修改产品
5. 点击删除图标删除产品

### 4. 管理订单
1. 点击侧边栏 "Orders"
2. 使用状态筛选或搜索
3. 点击眼睛图标查看详情
4. 使用下拉菜单更新订单状态

### 5. 管理优惠码
1. 点击侧边栏 "Coupons"
2. 查看所有优惠码卡片
3. 点击 "Add Coupon" 创建新优惠码
4. 点击 "Edit" 修改优惠码
5. 点击 "Delete" 删除优惠码

## 注意事项

### 演示模式
⚠️ 当前为演示模式，所有操作不会真正保存到数据库：
- 添加/编辑/删除操作会显示提示
- 刷新页面后数据恢复初始状态
- 需要连接真实后端 API 才能持久化数据

### 生产环境部署
要在生产环境使用，需要：
1. 实现真实的后端 API
2. 连接数据库（如 Supabase）
3. 实现真实的认证系统（JWT/Session）
4. 添加权限控制
5. 实现数据验证
6. 添加错误处理
